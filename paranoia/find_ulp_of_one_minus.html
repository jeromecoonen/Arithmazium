find_ulp_of_one_minus()

This function operates in parallel, but a factor of \(1 / B\) smaller than its sibiling find_ulp_of_one_plus just covered.

We start with the value \(2 / 3\), which rounds in any radix not a multiple of \(3\). It's rounding error is in ulps of values just less than 1.0. Values sixth and third share the rounding direction of TWO / THREE. But the negative value x has the opposite direction of magnitude rounding because of the change of sign.

To explore the algebra a bit, if sixth has the value \(1/6 + \epsilon\) then x has the value \(-1/6 + 2 \epsilon\) or \(- ( 1/6 - 2 \epsilon )\). That is, if sixth rounds up in magnitude then x rounds down, and vice versa.

As before, we take the larger choice of starting values for x. 

def find_ulp_of_one_minus(guess):
    """Compute u such at 1-u is the next smaller value than 1."""
    x = TWO / THREE
    sixth = x - ONE_HALF
    third = sixth + sixth
    x = third - ONE_HALF
    x = fabs(x + sixth)
    if (x < guess):
        x = guess
    # Now  x == (unknown no.) ulps of 1-.
