find_ulp_of_one_plus()

Herein lies one of the splendid numerical idioms of Paranoia. We'll encounter this technique in several functions to follow. The idea is this:

When searching for the smallest value with a given property, start with any handy value at least as big and iterate downward.

The code starts with the value \(4 / 3\), which rounds in any radix not a multiple of \(3\). It's rounding error is in ulps of 1.0. Subtracting from ONE_HALF gives sixth a rounding error in the opposite direction. If one is rounded up, the other is rounded down. Subtracting the two approximations of \(1 / 3\) sums the two error magnitudes. This binary diagram shows the case of even precision.

The result of the fiddling is a handful of ulps of \(1\). To give some headroom, Paranoia takes the larger of the the rounding noise x and the guess passed as a parameter. The function fails if the u is smaller than one ulp of \(1\). 

def find_ulp_of_one_plus(guess):
    """Compute u such at 1+u is the next larger value than 1."""
    x = FOUR / THREE    # rounds in ulps of 1
    third = x - ONE
    sixth = ONE_HALF - third
    x = sixth + sixth
    x = fabs(x - third)   # sums two rounding errors
    if x < guess:
        x = guess
    # Now x = (unknown no.) ulps of 1+.

The next iteration exhibits another idiom: the creation of nonzero low-order digits to encourage an upward round, if there's nearly a half-way case. The binary diagram below shows how u is halved, but with some nonzero low-order bits. Since u is some ulps of \(1.0\), u * u is some ulps of u. The factor THIRTY_TWO shifts those bits well back into u in radix 2, 8, or 16. The diagram continues the binary tale.

In decimal, the factor ensures some nonzero low-order digits in x. 

    while True:
        u = x
        x = ONE_HALF * u + THIRTY_TWO * u * u
        x = x + ONE
        x = x - ONE
        if (u <= x) or (x <= ZERO): break
    return u
